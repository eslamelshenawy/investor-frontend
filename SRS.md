# وثيقة مواصفات متطلبات البرمجيات (SRS)

## مشروع: رادار المستثمر (Investor Radar)

**الإصدار:** 2.0  
**التاريخ:** 31 ديسمبر 2025  
**الحالة:** شاملة ودقيقة (Enterprise Grade)

---

## 1. المقدمة (Introduction)

### 1.1 الغرض (Purpose)

تحدد هذه الوثيقة المتطلبات الوظيفية وغير الوظيفية لمنصة "رادار المستثمر"، وهي منصة ذكاء أعمال (BI) متقدمة مخصصة لتحليل البيانات الاقتصادية والاستثمارية في المملكة العربية السعودية، مدعومة بالذكاء الاصطناعي لتسهيل اتخاذ القرار للمستثمرين والخبراء.

### 1.2 نطاق المشروع (Project Scope)

المنصة عبارة عن تطبيق ويب تفاعلي (SPA) يجمع بين عرض البيانات الرسمية، التحليل الذكي، والتغذية الإخبارية التفاعلية. تهدف إلى سد الفجوة بين البيانات الخام والرؤى الاستثمارية القابلة للتنفيذ.

### 1.3 التعريفات والمصطلحات (Definitions & Acronyms)

- **Widget:** وحدة عرض بيانية (رسم بياني، KPI، أو جدول).
- **Dashboard:** لوحة تجميعية تحتوي على عدة Widgets.
- **Signal:** إشارة تحليلية ناتجة عن تغير في البيانات أو تحليل AI.
- **Feed:** واجهة عرض المحتوى المتجدد (مثل منصات التواصل).
- **Gemini AI:** محرك الذكاء الاصطناعي المستخدم لتحليل البيانات وتوليد النصوص.

## 2. الوصف العام (Overall Description)

### 2.1 منظور المنتج (Product Perspective)

يعمل "رادار المستثمر" كنظام مركزي يربط بين مصادر البيانات الحكومية (API Integration) والجهات الاستشارية، مع توفير مساحة تخصيص كاملة للمستخدم النهائي.

### 2.2 وظائف النظام (System Functions)

1.  **لوحات البيانات (Dashboards):** عرض مؤشرات الاقتصاد الكلي والقطاعي.
2.  **الذكاء الاصطناعي (AI Insights):** تحليل تلقائي للمخططات البيانية.
3.  **تغذية المحتوى (Modern Feed):** عرض الأخبار، البودكاست، والإنفوجرافيك.
4.  **إدارة الإشارات (Signals Management):** تتبع الفرص والمخاطر.
5.  **تخصيص المستخدم (Customization):** بناء لوحات خاصة واختيار مؤشرات مفضلة.

### 2.3 فئات المستخدمين (User Classes)

- **المستخدم القياسي (Standard):** استعراض البيانات العامة والاشتراك في التنبيهات.
- **المحلل (Analyst):** الوصول لبيانات أعمق وتصدير التقارير.
- **الخبير (Expert):** القدرة على إضافة رؤى (Insights) يدوية للنظام.
- **المدير (Admin):** إدارة النظام، ربط ملفات البيانات، ومراقبة سجلات التحديث.

## 3. المتطلبات الوظيفية (Functional Requirements)

### 3.1 نظام لوحات البيانات (Dashboard System)

- **[REQ-1.1]** يجب أن يدعم النظام لوحات بيانات "رسمية" (تثبت من قبل الإدارة) ولوحات "خاصة" (ينشئها المستخدم).
- **[REQ-1.2]** يجب أن تتوفر مكتبة Widgets تشمل أنواع: (Line, Bar, Area, Pie, KPI, External Embed).
- **[REQ-1.3]** إمكانية إضافة Widgets خارجية عبر بروتوكول Metabase أو روابط URL مباشرة (للمحللين والمدراء فقط).
- **[REQ-1.4]** دعم ميزة "المفضلة" (Starring) للوصول السريع للوحات من القائمة الجانبية.

### 3.2 محرك الذكاء الاصطناعي (AI Engine)

- **[REQ-2.1]** تكامل مع Google Gemini لتحليل بيانات الـ Widget بضغطة زر.
- **[REQ-2.2]** يجب أن يكون التحليل باللغة العربية، احترافياً، ولا يتجاوز 100 كلمة.
- **[REQ-2.3]** دعم ميزة "التفكير العميق" (Thinking Mode) للحصول على تحليلات اقتصادية معقدة.

### 3.3 مركز الاكتشاف والتغذية (Discovery Feed)

- **[REQ-3.1]** عرض محتوى متنوع يشمل: (بودكاست صوتي، ألبومات صور، إنفوجرافيك، مقالات).
- **[REQ-3.2]** ميزة "حرارة السوق" (Market Pulse) لعرض زخم القطاعات.
- **[REQ-3.3]** نظام التفاعل: الإعجاب، المشاركة، والحفظ.
- **[REQ-3.4]** عرض "ملخصات أسبوعية" و "حقائق تقنية" بشكل آلي.

### 3.4 سجل التغييرات والإشارات (Timeline & Signals)

- **[REQ-4.1]** توفير سجل زمني لجميع تحديثات البيانات (Data Freshness).
- **[REQ-4.2]** حساب "درجة التأثير" (Impact Score) لكل إشارة سوقية (من 1 إلى 100).
- **[REQ-4.3]** ربط الإشارات بـ Widgets معينة لتسهيل الانتقال من الخبر إلى الرقم.

## 4. متطلبات الواجهة الخارحية (Interface Requirements)

### 4.1 واجهة المستخدم (UI)

- اعتماد تصميم **Premium Dark/Light Hybrid** باستخدام Tailwind CSS.
- دعم كامل للغة العربية (RTL) مع خطوط عصرية (Inter/Outfit).
- واجهة متجاوبة بالكامل (Responsive) مع "شريط تنقل سفلي" (Bottom Nav) للهواتف.

### 4.2 واجهات البرمجيات (Software Interfaces)

- **الخلفية:** دعم أي Backend API (NestJS/Laravel) عبر RESTful Services.
- **تكامل AI:** Google Generative AI SDK.
- **التحليلات:** Metabase Embedded SDK.

---

## 5. المتطلبات غير الوظيفية (Non-functional Requirements)

### 5.1 الأداء (Performance)

- يجب ألا يتجاوز وقت تحميل الصفحة الأولى 2 ثانية.
- يجب أن تتم استجابة تحليل الـ AI في أقل من 5 ثوانٍ.

### 5.2 الأمان (Security)

- إدارة الصلاحيات (RBAC) بناءً على الأدوار المعرفة.
- حماية مفاتيح الـ API (Gemini/Metabase) عبر Environment Variables وبيئة خادم وسيطة.

### 5.3 الموثوقية (Reliability)

- توفير سجلات "Refresh Logs" لمراقبة فشل تحديث البيانات من المصادر الرسمية.
- معالجة الأخطاء (Error Handling) في حال تعطل خدمة AI أو عدم توفر الرابط الخارجي.

## 6. مصفوفة تتبع المتطلبات (Traceability Matrix) - ملامح تقنية

- **Routing:** React Router DOM (HashRouter).
- **State Management:** React Hooks (useState, useMemo, useEffect).
- **Data Structure:** TypeScript Interfaces (Defined in `types.ts`).
- **Visuals:** Recharts v3.5+.
